Estados:

      React no contiene los datos por lo cual los llama desde el backend, sin embargo al llamar esos datos va a necesitar almacenarlos en el navegador para mostrarlos en pantalla,
      eso es conocido como el estado de la aplicacion.

      El estado se llama de forma similar a los props, la forma de llamar el estado seria asi {this.state.title} {this.state.nTareas}, el constructor se pone antes del renderizado
      por que de esa forma se va a procesar toda la informacion necesaria para renderizar la vista

constructor(){}

      Es un metodo que se ejecuta inmediatamente el componente es creado, incluso antes del render(){}, al constructor se le puede decir que almacene los datos
      Nota: debajo del constructor se debe usar el super(); para que herede toda las funcionalidades de react (si no se pone sale un error)

      Se usa de la siguiente forma:

      class App extends Component {
        constructor(){
          super();
          this.state = {                                   *este es el estado del componente*
            title: "aplicacion de tareas",
            nTareas: 10
          }
        }
      }
